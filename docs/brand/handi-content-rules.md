# Handi Content Rules

## Core Principles

- Clarity beats novelty.
- Trust beats hype.
- One core message beats five weak ideas.
- Every asset should make the next step feel easy and safe.
- Every message should respect the real-world stress behind service needs.

## Clarity Rules

- Lead with the user problem or desired outcome.
- Keep sentences short and clean.
- Use concrete nouns and verbs.
- Prefer one CTA per asset.
- Remove filler that does not improve trust or understanding.
- If the copy needs a second read to make sense, simplify it.

## CTA Rules

Good CTA characteristics:

- Specific
- Calm
- Actionable
- Low-friction

Examples:

- Encuentra un pro
- Solicita ayuda
- Ver opciones
- Habla con un experto
- Activa tu perfil
- Empieza hoy
- Conoce como funciona

Avoid CTA styles like:

- Compra ya
- Aprovecha antes de que se acabe
- Da click urgente
- No te lo pierdas

## Rules For Ads

- Open with one clear benefit.
- Add one trust cue.
- Finish with one CTA.
- Do not overload the ad with features, channels, and proof all at once.
- Avoid miracle language and hard-sell countdown logic unless it is real and verifiable.

## Rules For Email

- Subject line must be clear before clever.
- First two lines should explain why the email matters.
- Body should move from context to value to action.
- Use short paragraphs and visible CTA language.
- Do not make the email sound like a mass blast if the message is meant to feel guided.

## Rules For WhatsApp

- Sound like a helpful follow-up, not like spam.
- Start with context so the person knows why they are receiving it.
- Keep the message skimmable.
- Make the next action obvious.
- Use emojis only if they help scanning and never more than one or two.

## Rules For Push

- Keep it tight.
- Make it relevant to the trigger.
- Focus on urgency only when the urgency is real.
- Avoid generic reminder copy with no value.
- Mention the action or benefit, not both in a crowded way.

## Rules For Landing Copy

- Hero should answer: what is this, why trust it, what should I do next?
- Support sections should reinforce trust, process, and proof.
- Match claims with tangible explanation.
- Keep headlines simple and benefit-first.
- Avoid long blocks that bury the point.

## Anti-Spam Rules

- Never fake scarcity.
- Never over-message the same CTA in one asset.
- Never sound like a lead farm.
- Never use excessive caps, punctuation, or urgency.
- Never hide who the message is for.
- Never imply unsafe certainty in regulated or operationally sensitive topics.

## Trust Rules

- If you mention safety, support it with process or context.
- If you mention quality, tie it to expert selection, professionalism, or clear workflow.
- If you mention speed, avoid promising instant resolution unless that path truly exists.
- If you mention price, do not imply cheapest-by-default positioning.

## Reusable Handi Checks

Before approving copy, ask:

- Is the audience obvious?
- Is the promise specific?
- Is the CTA easy to understand?
- Does this sound trustworthy?
- Would this make sense even without the brand logo?
- Does it feel like Handi, not generic marketplace copy?
